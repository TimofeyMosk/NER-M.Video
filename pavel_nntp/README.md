# Папка `pavel_nntp`

Здесь находятся материалы по анализу `query_clicks`, проектированию обработки данных и экспериментам с нормализацией поисковых запросов.

## С чего начать

1. `data_processing_pipeline_final.md` — короткая актуальная версия согласованного пайплайна.
2. `query_clicks_analysis.html` — основной отчет по датасету `query_clicks`.
3. `data_processing_story.html` — наглядное объяснение обработки данных на примерах.

## Содержание

| Файл | Краткое описание |
|---|---|
| `README.md` | Описание содержимого папки и назначение файлов. |
| `data_processing_pipeline_final.md` | Короткий итоговый пайплайн: очистка текста, ручной словарь стандартных названий, обработка кликов и дублей, агрегация, фильтрация и разбиение `train/valid`. Это основная актуальная версия. |
| `data_processing_pipeline.md` | Подробная техническая спецификация пайплайна с правилами, структурами данных, порогами качества и проверками. Используется как расширенный рабочий документ. |
| `data_processing_story.html` | Наглядная HTML-презентация о том, как сырые поисковые логи превращаются в обработанный датасет. Подходит для демонстрации друзьям или команде. |
| `query_clicks_analysis.html` | Готовый HTML-отчет с общей статистикой, качеством данных, дублями, популярными запросами, брендами, SKU, ценами и позициями. |
| `query_clicks_report.py` | Скрипт, который читает `query_clicks.parquet`, рассчитывает статистику и создает `query_clicks_analysis.html`. |
| `process_query_clicks.py` | Полный воспроизводимый пайплайн обработки: каталог, нормализация, клики, дубли, агрегация и разбиение `train/valid`. Все результаты сохраняет в `data/` внутри этой папки. |
| `test_process_query_clicks_smoke.py` | Сквозной smoke-тест пайплайна на небольшом искусственном наборе данных. |
| `export_normalization_sample.py` | Создает воспроизводимую выборку из 1000 уникальных запросов в CSV и HTML в папке `data/reports/`. |
| `query_normalization_100.html` | HTML-таблица со 100 реальными запросами в формате «оригинал → результат обработки». |
| `query_normalization_100.csv` | Те же 100 примеров нормализации в CSV для просмотра и дальнейшей обработки. |
| `query_normalization_sample.py` | Скрипт воспроизводимой выборки 100 запросов и применения согласованных правил очистки текста. |
| `mine_russian_product_aliases.py` | Экспериментальный скрипт поиска русскоязычных вариантов английских брендов и товарных названий по кликам. Не входит в финальный пайплайн. |
| `russian_product_alias_candidates.csv` | Сырые кандидаты, найденные экспериментальным поиском алиасов. Содержат ложные совпадения и требуют ручной проверки. |
| `russian_product_aliases.json` | Экспериментальный автоматически собранный словарь. Не следует использовать как готовый рабочий маппинг без очистки. |
| `russian_product_alias_report.html` | HTML-отчет по экспериментальному поиску алиасов с частотами, уверенностью и примерами запросов. |
| `empty.txt` | Пустой файл-заглушка, добавленный при создании папки. |

## Обработанные данные

Папка `data/` содержит результаты полного запуска пайплайна: Parquet-таблицы, словари, конфигурацию, manifest и HTML-отчет. Она добавлена в `.gitignore`, потому что содержит крупные генерируемые файлы.

## Исходные данные

Скрипты используют файлы из соседней папки `../cu_ws/`:

```text
query_clicks.parquet
sku_desc.parquet
skus.pkl
```

Папка `cu_ws` и архив с исходными данными добавлены в `.gitignore`, поэтому большие файлы не попадают в Git.

## Служебные файлы

Папка `__pycache__/` создается Python автоматически. Она не содержит исходный код или результаты анализа и может быть пересоздана при запуске скриптов.
