# Высокоуверенная silver-разметка

Папка содержит автоматическую разметку 15% предобработанного корпуса как рабочую validation-выборку.

Важно: это **silver**, а не ручной gold. Категория строится по согласованным кликам, явные сущности — по точному токенному совпадению с брендами, моделями и характеристиками каталога. Не найденные сущности не считаются гарантированно отсутствующими.

## Запуск

```powershell
.venv\Scripts\python.exe silver_annotation\build_silver_dataset.py --percent 0.15
.venv\Scripts\python.exe silver_annotation\verify_silver_dataset.py
```

По умолчанию создаётся 15% от итогового предобработанного корпуса и пять shards. Все строки помечаются `split=validation`. Уровни A/B сохраняют строгие пороги (`confidence >= 0.90`, не менее 5 кликов), а недостающий объём добирается уровнем C (`confidence >= 0.80`, не менее 3 кликов).

## Выходы

- `output/silver_labels.parquet` — единый датасет.
- `output/shards/part-*.parquet` — пять частей для ограниченной памяти.
- `output/annotation_metrics.json` — контрольные метрики.
- `output/manual_review_sample.csv` — 1000 примеров для ручной проверки.
- `output/ANNOTATION_REPORT.md` — описание методики, результатов и ограничений.

Категорийные метки пригодны для обучения классификатора. Entity-разметка является частичной: положительные spans высокоуверенные, а `bio_supervision_mask=false` на остальных токенах запрещает автоматически трактовать их как надёжный `O`.
